# 人事評価・研修統合プラットフォーム 要件定義書

## 1. ドキュメント概要

### 1.1 目的
本ドキュメントは、人事評価・研修統合プラットフォームの機能要件および非機能要件を定義し、開発チームとステークホルダー間の共通認識を形成することを目的とします。

### 1.2 対象読者
- プロジェクトマネージャー
- 開発チーム（エンジニア、デザイナー）
- 品質保証チーム
- ステークホルダー（一般社団法人人材育成支援機構）

### 1.3 用語定義

| 用語 | 定義 |
|------|------|
| LMS | Learning Management System（学習管理システム） |
| MBO | Management by Objectives（目標管理制度） |
| OKR | Objectives and Key Results |
| SCORM | Sharable Content Object Reference Model |
| MVP | Minimum Viable Product（最小限の実用的な製品） |

---

## 2. システム概要

### 2.1 システムの目的
人事評価制度と研修管理システムを統合し、評価結果に基づく最適な研修推奨と、研修成果の評価への反映を実現することで、従業員の自律的なスキルアップと組織全体のパフォーマンス向上を支援する。

### 2.2 ユーザー種別

| ユーザー種別 | 説明 | 主な利用機能 |
|--------------|------|--------------|
| システム管理者 | プラットフォーム全体の管理・設定を行う | テナント管理、システム設定、監査ログ |
| テナント管理者 | 企業・組織単位での管理を行う | 組織設定、ユーザー管理、評価制度設定 |
| 人事担当者 | 人事評価・研修の運用管理を行う | 評価サイクル管理、研修計画、レポート |
| 管理職（評価者） | 部下の評価・育成を行う | 評価実施、目標承認、進捗確認 |
| 一般従業員（被評価者） | 自己評価・研修受講を行う | 目標設定、自己評価、研修受講 |

## 3. 機能要件

### 3.1 認証・認可システム

#### FR-AUTH-001: ユーザー認証
- **説明**: ユーザーがシステムにログインする機能
- **詳細要件**:
  - メールアドレス/パスワードによる認証
  - 多要素認証（MFA）のサポート（TOTP、SMS）
  - SSO連携（SAML 2.0、OpenID Connect）
  - パスワードポリシーの設定（長さ、複雑性、有効期限）
  - ログイン試行回数制限とアカウントロック
  - パスワードリセット機能

#### FR-AUTH-002: ロールベースアクセス制御（RBAC）
- **説明**: ユーザーの役割に応じたアクセス権限管理
- **詳細要件**:
  - 階層的なロール定義
  - 機能単位での権限設定
  - データアクセス範囲の制御（部署、職位など）
  - 一時的な権限付与（代理承認など）

#### FR-AUTH-003: セッション管理
- **説明**: ユーザーセッションの管理
- **詳細要件**:
  - セッションタイムアウト設定
  - 同時ログイン制御
  - セッション無効化（強制ログアウト）
  - アクティビティログ

---

### 3.2 人事評価システム

#### FR-EVAL-001: 評価制度設定
- **説明**: 組織の評価制度を設定する機能
- **詳細要件**:
  - 評価手法の選択（MBO、OKR、コンピテンシー評価、360度評価）
  - 評価期間・サイクルの設定
  - 評価項目・配点のカスタマイズ
  - 評価尺度の定義（5段階、7段階など）
  - 評価ワークフローの設定

#### FR-EVAL-002: 目標設定・管理
- **説明**: 従業員の目標を設定・管理する機能
- **詳細要件**:
  - SMART目標の設定支援
  - 目標の階層化（会社→部門→個人）
  - 目標のウェイト設定
  - 目標の承認ワークフロー
  - 目標の進捗更新
  - 期中での目標変更管理

#### FR-EVAL-003: 評価実施
- **説明**: 評価を実施する機能
- **詳細要件**:
  - 自己評価入力
  - 上司評価入力
  - 360度評価（同僚、部下からの評価）
  - 評価コメント入力
  - 評価の一時保存
  - 評価の提出・承認ワークフロー

#### FR-EVAL-004: 評価結果管理
- **説明**: 評価結果を管理・分析する機能
- **詳細要件**:
  - 評価結果の集計・計算
  - 評価分布の確認
  - 評価履歴の管理
  - 評価結果のフィードバック機能
  - 評価結果のエクスポート（CSV、PDF）

#### FR-EVAL-005: 1on1ミーティング管理
- **説明**: 上司と部下の1on1ミーティングを管理
- **詳細要件**:
  - ミーティングスケジュール管理
  - アジェンダ・議事録の記録
  - アクションアイテムの追跡
  - ミーティング履歴の参照

### 3.3 研修管理システム（LMS）

#### FR-LMS-001: コンテンツ管理
- **説明**: 研修コンテンツを管理する機能
- **詳細要件**:
  - 動画コンテンツのアップロード・管理（MP4、WebM）
  - PDFドキュメントの管理
  - SCORM 1.2/2004パッケージのサポート
  - コンテンツのバージョン管理
  - コンテンツのカテゴリ・タグ付け
  - コンテンツの公開・非公開設定

#### FR-LMS-002: コース管理
- **説明**: 研修コースを作成・管理する機能
- **詳細要件**:
  - コースの作成（複数コンテンツの組み合わせ）
  - 前提条件の設定（必須コースの完了など）
  - 受講期限の設定
  - コースの難易度設定
  - 対象者の設定（職種、職位、部署など）
  - コースの公開スケジュール管理

#### FR-LMS-003: 受講管理
- **説明**: 従業員の受講状況を管理する機能
- **詳細要件**:
  - 個人への研修割り当て
  - グループへの一括割り当て
  - 受講進捗の追跡
  - 動画視聴履歴の記録（視聴時間、一時停止、巻き戻しなど）
  - 受講完了条件の設定
  - 受講リマインダー通知

#### FR-LMS-004: テスト・アンケート機能
- **説明**: 研修の理解度確認と効果測定
- **詳細要件**:
  - 多様な問題形式（選択式、記述式、穴埋め）
  - 問題のランダム出題
  - 制限時間の設定
  - 合格基準の設定
  - 再受験の許可設定
  - アンケートの作成・集計

#### FR-LMS-005: 修了証・資格管理
- **説明**: 研修修了の証明と資格管理
- **詳細要件**:
  - 修了証の自動発行
  - 資格・認定の管理
  - 有効期限の管理
  - 更新研修の通知

### 3.4 連携機能

#### FR-INT-001: 評価結果に基づく研修レコメンド
- **説明**: 評価結果から推奨研修を提示する機能
- **詳細要件**:

  - 推奨理由の表示
  - 推奨研修の優先度設定

#### FR-INT-002: 研修履歴の評価反映
- **説明**: 研修受講状況を評価に反映する機能
- **詳細要件**:
  - 研修受講履歴の自動取り込み
  - テスト成績の評価参考情報化
  - 学習時間の可視化
  - 評価シートへの自動挿入

#### FR-INT-003: スキルマップ管理
- **説明**: 従業員のスキルを可視化する機能
- **詳細要件**:
  - スキル項目の定義
  - スキルレベルの設定
  - 現状スキルの評価
  - 目標スキルの設定
  - スキルギャップの可視化
  - スキル習得計画の作成

#### FR-INT-004: キャリアパス管理
- **説明**: キャリア開発を支援する機能
- **詳細要件**:
  - キャリアパスの定義
  - 必要スキル・研修の紐付け
  - 現在位置の表示
  - 次のステップへの道筋表示

## 4. 非機能要件

### 4.1 性能要件

| 項目 | 要件 |
|------|------|
| 同時接続ユーザー数 | 1,000ユーザー以上 |
| ページ応答時間 | 95%のリクエストが2秒以内 |
| API応答時間 | 95%のリクエストが500ms以内 |
| 動画ストリーミング | 1080p品質での安定配信 |
| データベースクエリ | 95%が100ms以内 |
| バッチ処理 | 1万件の評価集計を10分以内 |

### 4.2 可用性要件

| 項目 | 要件 |
|------|------|
| システム稼働率 | 99.5%以上（年間ダウンタイム43.8時間以内） |
| 計画メンテナンス | 月1回、4時間以内 |
| 障害復旧時間（RTO） | 4時間以内 |
| 復旧ポイント（RPO） | 1時間以内（データ損失） |
| バックアップ | 日次フルバックアップ、時間次増分バックアップ |

### 4.3 セキュリティ要件

#### 4.3.1 通信セキュリティ
- すべての通信をTLS 1.3で暗号化
- HSTS（HTTP Strict Transport Security）の適用
- 証明書の自動更新

#### 4.3.2 データセキュリティ
- データベースの暗号化（AES-256）
- 機密データの暗号化保存（評価結果、個人情報）
- データマスキング機能
- 安全なデータ削除

#### 4.3.3 認証・認可セキュリティ
- 多要素認証（MFA）の必須化オプション
- パスワードのハッシュ化（bcrypt、salt）
- セッショントークンの安全な管理
- IPアドレス制限
- ブルートフォース攻撃対策

#### 4.3.4 アプリケーションセキュリティ
- OWASP Top 10への対策
  - SQLインジェクション対策
  - XSS（クロスサイトスクリプティング）対策
  - CSRF（クロスサイトリクエストフォージェリ）対策
- 入力値のバリデーション
- 出力のエスケープ処理
- セキュリティヘッダーの適用

#### 4.3.5 運用セキュリティ
- 定期的な脆弱性診断（四半期ごと）
- ペネトレーションテスト（年1回）
- セキュリティパッチの適用（72時間以内）
- インシデント対応計画の策定

### 4.4 スケーラビリティ要件

- 水平スケーリング対応（ステートレスアーキテクチャ）
- マイクロサービスによる個別スケーリング
- 自動スケーリング（負荷に応じた自動調整）
- 将来的な拡張を考慮した設計（10倍の成長に対応）

### 4.5 保守性要件

- コードの可読性（コーディング規約の遵守）
- 自動テストのカバレッジ80%以上
- CI/CDパイプラインによる自動デプロイ
- ドキュメントの整備
- ログの標準化と一元管理
- 監視・アラート体制の構築

### 4.6 互換性要件

#### 4.6.1 ブラウザ対応
- Google Chrome（最新2バージョン）
- Mozilla Firefox（最新2バージョン）
- Microsoft Edge（最新2バージョン）
- Safari（最新2バージョン）

#### 4.6.2 デバイス対応
- デスクトップPC（Windows、macOS）
- タブレット（iPad、Android）
- スマートフォン（iOS、Android）- 閲覧のみ

#### 4.6.3 外部システム連携
- SSO（SAML 2.0、OpenID Connect）
- 人事システムとのAPI連携
- Slack/Microsoft Teams連携
- カレンダー連携（Google Calendar、Outlook）

### 4.7 法令・コンプライアンス要件

- 個人情報保護法への準拠
- GDPR対応（将来的な海外展開を考慮）
- データ保持期間の管理（法定保存期間の遵守）
- 監査証跡の保持（最低5年間）
- アクセス権限の定期見直し

## 5. システム境界

### 5.1 システム範囲内
- 人事評価機能
- 研修管理機能（LMS）
- 評価と研修の連携機能
- 分析・レポート機能
- ユーザー管理・認証機能
- 通知機能

### 5.2 システム範囲外
- 給与計算システム
- 勤怠管理システム
- 経費精算システム
- 採用管理システム（ATS）

ただし、上記システムとのAPI連携は将来的な拡張として検討する。

## 6. 制約事項

### 6.1 技術的制約
- AWSクラウド環境での構築
- マイクロサービスアーキテクチャの採用
- コンテナ技術（Docker、ECS）の使用

### 6.2 ビジネス制約
- 開発期間：要件定義開始から正式リリースまで約10ヶ月
- 予算：別途見積もり
- MVP機能の優先度に基づく段階的リリース

### 6.3 法的制約
- 個人情報の取り扱いに関する法令遵守
- データセンターの国内設置

## 7. 前提条件

- クライアント企業は既存の人事評価制度を持っている
- 研修コンテンツは動画を中心に提供される
- インターネット接続環境が整っている
- ユーザーは基本的なPCスキルを持っている

## 8. 優先度定義

### MVP（Phase 1）で実装する機能
1. 基本的な認証・認可
2. 目標設定・評価実施（MBO）
3. 動画コンテンツの配信・受講管理
4. 基本的なレポート機能
5. 評価結果と研修履歴の基本連携

### Phase 2で実装する機能
1. 360度評価
2. SCORM対応
3. 高度な分析・レコメンド機能
4. スキルマップ管理
5. 外部システム連携（SSO、Slack等）

### Phase 3以降で検討する機能
1. AI活用機能
2. キャリアパスシミュレーション
3. 高度なパーソナライゼーション
